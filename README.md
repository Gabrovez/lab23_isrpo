# Крестики-нолики (Tic-Tac-Toe)

## 1. Описание игры

- **Правила игры**: Игра происходит на поле 3×3. Два игрока по очереди ставят символы «X» и «O» в свободные клетки. Первым ходит игрок «X».

- **Логика победы/поражения**: Победа присуждается игроку, который первым соберёт линию из трёх своих символов по горизонтали, вертикали или диагонали. Если все клетки заполнены, но победитель не выявлен, фиксируется ничья.

- **Особенности реализации**: Приложение разработано с использованием HTML для структуры, CSS для стилизации сетки и элементов, JavaScript для логики игры.

## 2. Игровая механика

- **Как определяется победитель?**: После каждого хода запускается функция проверки результата. Она анализирует текущее состояние поля на наличие выигрышных комбинаций.

- **Как обрабатываются клики?**: Обработчик события `handleCellClick` фиксирует нажатие на клетку. Проверяется, активна ли игра и пуста ли клетка. Если условия выполнены, в клетку записывается символ текущего игрока.

- **Как работает перезапуск?**: При нажатии на кнопку перезапуска вызывается функция `restartGame`. Она сбрасывает переменные состояния, очищает игровое поле от символов и передаёт ход игроку «X».

## 3. Структура данных

- **Объяснить массив board**: Переменная `board` представляет собой массив из 9 строк. Каждый элемент соответствует ячейке игрового поля. Изначально массив заполнен пустыми строками.

- **Объяснить условия победы**: Выигрышные комбинации хранятся в массиве массивов. Каждый вложенный массив содержит индексы клеток (например, [0, 1, 2]), образующих линию. Всего предусмотрено 8 комбинаций (3 горизонтали, 3 вертикали, 2 диагонали).

- **Как хранится состояние игры?**: 
  - `currentPlayer`: хранит символ текущего игрока («X» или «O»).
  - `gameActive`: булевый флаг, указывающий, продолжается ли игра (true) или завершена (false).

## 4. Алгоритмы

- **Алгоритм проверки победы**: Цикл перебирает все выигрышные комбинации. Для каждой комбинации проверяется, заполнены ли соответствующие клетки в массиве `board` одинаковым непустым символом.

- **Алгоритм определения ничьи**: Если победа не обнаружена, проверяется наличие пустых строк в массиве `board`. Если пустых клеток нет, игра заканчивается ничьей.

- **Алгоритм смены игрока**: Если результат хода не выявил победителя или ничью, значение переменной `currentPlayer` меняется на противоположное (с «X» на «O» или с «O» на «X»).
